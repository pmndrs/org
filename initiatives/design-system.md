# Design System

**Status: Active**

## Motivations

Pmndrs has always had a strong design language but over the years as individual members build documentation, widgets, websites and more the cohesion of that design language has frayed. It should be easier for devs, both in and out of the collective, to build within the pmndrs design language without needing a designer. This leads us to a design system that can be implemented both in code and with an AI agent.

## Goals

- Help anyone create projects that feel like pmndrs without a designer.
- Provide a shared design system for packages, docs, and websites.
- Make the pmndrs design language usable by AI agents.
- Preserve a cohesive collective identity while allowing individual projects to have their own character.

## Scope

Included:

- Unified controls, menu, panel, widget, articles, docs, OG image, etc.
- Text/copy guidelines.
- A waterfall visual style guide.
- AI skill (style guide in md format).
- Branding package with design system that can be imported.

Excluded:

- Auto-generating images on brand.
- Redesigning the pmndrs website.
- Sound design.
